# Mobile Inventory Management Application Skill File

## 1. Architecture & Stack Overview
- **Frontend Framework:** Ionic Framework (React or Angular wrapper recommended, currently drafted for Angular/React generic using web components)
- **Language:** TypeScript
- **Backend API:** Node.js (Express or NestJS) / Spring Boot (Java)
- **Database:** Oracle Database (Oracle 19c+ recommended for JSON support and advanced features)
- **Data Access:** TypeORM or Prisma (if Node.js) / Hibernate (if Spring Boot) connecting to Oracle.

## 2. Best Practices for Ionic + TypeScript
- **Component Reusability:** Create highly reusable UI components (e.g., `DeviceCard`, `StatusBadge`).
- **State Management:** Use Redux, NgRx, or Context API depending on the chosen underlying framework to manage inventory state (like device list, user session).
- **Service Layer:** Abstract all API calls to a `DeviceService` to handle `GET`, `POST`, `PUT` requests for devices.
- **Offline Capabilities:** Utilize Ionic Storage or SQLite for caching device lists so users can view data even when offline or in a dead zone.
- **Hardware Integration:** Use Capacitor for native features:
  - `@capacitor-community/barcode-scanner`: For scanning IMEI/Serial number barcodes.
  - `@capacitor/camera`: For taking photos of device condition during assignment/return.

## 3. Database Schema Recommendations (Oracle)
Since Oracle is the DB, we can leverage relational structures with foreign keys.

```sql
-- Devices Table
CREATE TABLE DEVICES (
    DEVICE_ID VARCHAR2(50) PRIMARY KEY,
    IMEI VARCHAR2(50) UNIQUE NOT NULL,
    SERIAL_NUMBER VARCHAR2(100) UNIQUE,
    MAKE VARCHAR2(50),
    MODEL VARCHAR2(100),
    COLOR VARCHAR2(30),
    OS_VERSION VARCHAR2(50),
    STATUS VARCHAR2(20) DEFAULT 'AVAILABLE', -- AVAILABLE, ASSIGNED, IN_REPAIR, LOST, RETIRED
    CREATED_AT TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Assignments Table
CREATE TABLE ASSIGNMENTS (
    ASSIGNMENT_ID NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    DEVICE_ID VARCHAR2(50) REFERENCES DEVICES(DEVICE_ID),
    ASSIGNED_TO_USER_ID VARCHAR2(50),
    PROJECT_ID VARCHAR2(50),
    PROGRAM_ID VARCHAR2(50),
    ASSIGNED_BY VARCHAR2(50),
    ASSIGNMENT_DATE TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    RETURN_DATE TIMESTAMP,
    ASSIGNMENT_TYPE VARCHAR2(20) -- INDIVIDUAL, PROJECT, PROGRAM
);
```

## 4. Key UI/UX Patterns
- **Dashboard:** A quick glance at statistics (Total, Assigned, Available, In Repair).
- **Device List:** Infinite scrolling list with search by IMEI/Serial.
- **Device Details:** A clean card showing current assignment and a history log.
- **Action Buttons:** Prominent Floating Action Button (FAB) or bottom bar for quick "Scan & Add" or "Scan & Reassign".

## 5. Development Phases
- **Phase 1:** Set up Ionic blank project (`ionic start MobileInventory blank --type=angular`). Build static UI components (Dashboard, List, Details).
- **Phase 2:** Set up backend APIs connecting to Oracle DB. Implement CRUD for `DEVICES` table.
- **Phase 3:** Integrate `@capacitor-community/barcode-scanner`. Connect frontend with backend APIs.
- **Phase 4:** Implement Assignment Logic and Reporting Dashboards.
